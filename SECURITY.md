# Security Policy

## Supported Versions

We release patches for security vulnerabilities. The following versions are currently supported:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability, please send an email to security@devguardian.ai

We appreciate all responsible disclosures and will work to quickly address any issues. Please include:

1. Description of the vulnerability
2. Steps to reproduce
3. Potential impact
4. Any suggested fixes (optional)

We aim to acknowledge reports within 48 hours and provide a timeline for remediation.

## Security Best Practices

- Keep your API keys and secrets confidential
- Use strong passwords
- Enable two-factor authentication when available
- Regularly update your dependencies
- Review security scan results in GitHub Actions

## Scope

This security policy applies to the DevGuardian AI codebase. Third-party integrations and services are governed by their own security policies.
